ADOC Chrome Extension

Deployment Guide

Version: 1.1.0 | Date: January 30, 2026
Document Type: Deployment Instructions

Overview

This guide provides step-by-step instructions to deploy the ADOC Chrome Extension in your organization. The extension integrates Acceldata's Data Observability Cloud with Power BI, providing real-time data reliability insights.

Important: Choose one deployment method based on your organization's requirements:
  • Internal Distribution - For enterprise/corporate environments (Recommended)
  • Manual Installation - For testing and small teams

Prerequisites

Requirement Details
Browser Google Chrome version 90 or later
Access Google Workspace Admin Console (for internal distribution)
Extension File ADOC-Extension-DEMO-v1.1.0.zip
User Credentials Valid Acceldata account at indiumtech.acceldata.app

Method 1: Internal Distribution (Google Workspace)

// This method is recommended for enterprise deployments with centralized management

Step 1: Prepare Extension Package

1.1 Locate the extension file:

ADOC-Extension-DEMO-v1.1.0.zip

// Ensure the zip file is downloaded and accessible on your computer

1.2 Verify file integrity:

  • File size should be approximately 41 KB
  • Extract and verify contents include: manifest.json, html/, css/, js/, icons/

Screenshot showing the extracted folder structure

Step 2: Access Google Workspace Admin Console

2.1 Open your browser and navigate to:

https://admin.google.com

2.2 Login with your Google Workspace administrator credentials

// You must have admin privileges to manage Chrome extensions

Google Workspace Admin Console login page

Step 3: Navigate to Chrome Management

3.1 From the Admin Console home page, click:

Devices → Chrome → Apps & Extensions

3.2 Click on "Settings" in the left sidebar

3.3 Ensure the following options are enabled:

  • ☑ Allow users to install other apps & extensions
  • ☑ Allow installation of specific extensions

Chrome Apps & Extensions settings page

Step 4: Upload Extension

4.1 In the Apps & Extensions page, click:

Add → Upload private extension

4.2 Click "Select file" and choose:

ADOC-Extension-DEMO-v1.1.0.zip

4.3 Wait for upload to complete (10-30 seconds)

// Do not close the browser during upload

Upload private extension dialog box

5.1 After upload, configure the following:

Setting Value
Extension Name ADOC Reliability Metrics

Allow/Block Allow

5.2 Click "Save"

6.1 In the "Applied to" section, choose:

  • All users - Deploy to entire organization

IT Team → Data Analytics Team → All Users

Method 2: Manual Installation (Testing/Small Teams)

// Use this method for development, testing, or small team deployments

Step 1: Extract Extension Files

1.1 Locate the extension file:

ADOC-Extension-DEMO-v1.1.0.zip

1.2 Extract the zip file:

  • Windows: Right-click → Extract All
  • Mac: Double-click the zip file
  • Linux: unzip ADOC-Extension-DEMO-v1.1.0.zip

1.3 You should see a folder named chrome-extension

[INSERT SCREENSHOT]
Extracted folder showing chrome-extension directory

Step 2: Open Chrome Extensions Page

2.1 Open Google Chrome browser

2.2 Navigate to extensions page (choose one method):

Method A: Type in address bar:

chrome://extensions/

Method B: Click menu:

⋮ Menu → More Tools → Extensions
[INSERT SCREENSHOT]
Chrome extensions page

Step 3: Enable Developer Mode

3.1 On the extensions page, find the toggle switch in the top right corner

3.2 Enable "Developer mode"

// Developer mode allows loading unpacked extensions

3.3 You'll see new buttons appear: "Load unpacked", "Pack extension", "Update"

[INSERT SCREENSHOT]
Developer mode toggle enabled with new buttons visible

Step 4: Load Extension

4.1 Click the "Load unpacked" button

4.2 In the file browser dialog:

  • Navigate to the extracted folder
  • Select the chrome-extension folder
  • Click "Select Folder" (Windows) or "Open" (Mac)

// Make sure you select the chrome-extension folder, not the parent folder

File browser showing chrome-extension folder selected

Step 5: Verify Installation

5.1 The extension should now appear in your extensions list:

Field Value
Name ADOC Reliability Metrics
Version 1.1.0
Status On (blue toggle)
Icon Dark blue "a" logo

5.2 Check for the extension icon in your Chrome toolbar (top right)

Success! Extension is installed and ready to use

Troubleshooting

Common Issues and Solutions

Issue Solution
Extension not appearing in toolbar
  • Refresh chrome://extensions/ page
  • Click puzzle icon and pin extension
  • Restart Chrome browser
Installation failed
  • Verify you selected the correct folder
  • Check manifest.json exists in selected folder
  • Ensure Developer mode is enabled
Login doesn't redirect back
  • Manually close the auth tab after login
  • Click extension icon again
  • Check for success notification
"No Assets Found" for all reports
  • Verify datasets are onboarded to ADOC
  • Check user has permissions in ADOC
  • Contact ADOC administrator
Extension crashes or freezes
  • Clear browser cache: chrome://settings/clearBrowserData
  • Remove and reinstall extension
  • Contact support with error details

Getting Detailed Error Information

For Users:

1. Right-click extension icon
2. Select "Inspect popup"
3. Click Console tab
4. Screenshot any error messages
5. Send to support@acceldata.io

// Error messages help support team diagnose issues quickly

Updates and Maintenance

Updating the Extension

For Google Workspace Deployment:

1. Download new version (e.g., ADOC-Extension-v1.2.0.zip)

2. Go to Admin Console → Chrome → Apps & Extensions

3. Find "ADOC Reliability Metrics"

4. Click "Update" button

5. Upload new zip file

6. Click "Save"

7. Update deploys automatically within 24 hours

// Users don't need to take any action for updates

For Manual Installation:

1. Go to chrome://extensions/

2. Find ADOC extension

3. Click "Remove"

4. Follow installation steps with new version

// Users may need to login again after update

Rollback Procedure

If issues occur with new version:

  1. Admin Console → Apps & Extensions
  2. Find ADOC extension
  3. Click "Version history"
  4. Select previous stable version
  5. Click "Rollback"
  6. Confirm rollback

// Rollback takes effect within 1-2 hours

Security and Compliance

Security Features

  • Encryption: AES-256-GCM for credential storage
  • Storage: Local browser only (no third-party transmission)
  • Permissions: Minimal required permissions only
  • HTTPS: All API calls use encrypted connections
  • Authentication: Leverages ADOC platform security

Data Privacy

Important Information:
  • Extension does NOT collect user analytics
  • Extension does NOT share data with third parties
  • Extension does NOT track user behavior
  • All data stored locally in encrypted format
  • Data cleared completely on logout
